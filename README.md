Проект состоит из двух независимых компонентов:

- **proxy_harvester.py** — собирает, проверяет и обновляет список прокси.
- **proxy_forwarder.py** — поднимает локальный SOCKS5-сервер и маршрутизирует трафик через лучший доступный прокси.

`proxy_harvester.py` периодически собирает список публичных прокси, проверяет их доступность, измеряет задержку и сохраняет актуальное состояние в `proxies/state.json`. `proxy_forwarder.py` использует этот файл для выбора лучшего доступного прокси заданной страны, после чего поднимает локальный SOCKS5-сервер на указанном порту. Во время работы форвардер регулярно проверяет состояние текущего upstream-прокси, автоматически переключается на следующий доступный при сбое и перечитывает `state.json`, благодаря чему список прокси может обновляться без перезапуска сервиса.
